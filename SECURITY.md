# Безопасность

## Сообщение об уязвимостях

Если вы обнаружили уязвимость в безопасности, пожалуйста, **не создавайте публичный issue**. Вместо этого сообщите нам по электронной почте в приватном порядке.

### Как сообщить об уязвимости

1. Отправьте письмо на адрес безопасности проекта
2. Включите:
   - Описание уязвимости
   - Шаги для воспроизведения
   - Возможное влияние
   - Предлагаемое исправление (если есть)

3. Даём ответ в течение 48 часов
4. Мы будем работать над исправлением и выпустим патч

## Рекомендации безопасности

### Для frontend (Flutter)

- ✅ Никогда не сохраняйте чувствительные данные в SharedPreferences без шифрования
- ✅ Используйте flutter_secure_storage для хранения токенов
- ✅ Проверяйте целостность SSL сертификатов
- ✅ Используйте HTTPS для всех API запросов

### Для backend (Django)

- ✅ Установите `DEBUG=False` в production
- ✅ Используйте сильные SECRET_KEY
- ✅ Включите HTTPS (SECURE_SSL_REDIRECT=True)
- ✅ Установите CSRF и CORS правильно
- ✅ Используйте environment variables для чувствительных данных
- ✅ Регулярно обновляйте зависимости

### Для базы данных

- ✅ Используйте сильные пароли
- ✅ Ограничьте доступ на уровне сети
- ✅ Включите аутентификацию
- ✅ Регулярно создавайте резервные копии
- ✅ Используйте SSL для подключений

## Обновления безопасности

Если вам нужно срочно обновить зависимости:

```bash
# Backend
pip install -U pip setuptools
pip list --outdated
pip install -r requirements.txt --upgrade

# Frontend
flutter pub upgrade
flutter pub outdated
```

## Тестирование безопасности

```bash
# Backend: проверка известных уязвимостей
pip install safety
safety check

# Анализ кода
pip install bandit
bandit -r backend/

# Frontend
flutter pub global activate patrol_cli
```
